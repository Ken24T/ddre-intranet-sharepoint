# Intranet UI – First‑time User Guide

This guide explains how to navigate the intranet interface from the perspective of a first‑time user. It covers the layout, navigation surfaces, and the purpose of each UI area.

## Quick orientation

The intranet UI is split into five main areas:
1. **Navbar** (top) – global controls and quick access.
2. **Sidebar** (left) – hub navigation and favourites.
3. **Content area** (centre) – dashboards, cards, and app views.
4. **Status bar** (bottom) – system status and notifications.
5. **Panels and flyouts** – context panels for tasks, settings, and notifications.

> [!suggestion] What you should see
> - A top bar with the site title and actions.
> - A left navigation rail with hub entries.
> - A large central content area with cards or app content.
> - A slim status bar across the bottom.

## Navbar (top bar)

The navbar contains global actions and quick access tools:
- **Site title** and hub branding.
- **Search** for content and people.
- **Tasks** button (opens the Tasks panel).
- **Notifications** bell (opens notification flyout).
- **Help Centre** button.
- **User profile menu** (settings and preferences).

### Searching
1. Select the search field in the navbar.
2. Enter a search term and press **Enter**.
3. Select a result to open it.

> [!suggestion] What you should see
> - A dropdown with results as you type.
> - A full search results page when you submit.

### Tasks
1. Select the **Tasks** icon.
2. Manage tasks in the slide‑out panel.

> [!suggestion] What you should see
> - A Tasks panel with list, filters, and quick actions.

### Notifications
1. Select the **Notifications** bell.
2. Review grouped alerts (overdue, due today).
3. Select a notification to open tasks.

> [!suggestion] What you should see
> - A notification flyout with grouped items.
> - A badge count on the bell when there are unread items.

### Help Centre
1. Select **Help Centre** to open the hub.
2. Browse or search help content.

> [!suggestion] What you should see
> - Help Centre cards and filters.
> - A search field dedicated to help content.

### User profile menu
The profile menu provides:
- **Theme toggle** (light/dark).
- **Settings** panel entry.
- **Help** links.

> [!suggestion] What you should see
> - A menu with theme and settings options.

## Sidebar (left navigation)

The sidebar lists available hubs and favourites.

### Hub navigation
1. Select any hub to switch the main content area.
2. The active hub highlights in the sidebar.

> [!suggestion] What you should see
> - The content area refreshes to the selected hub.
> - The active hub is visually highlighted.

### Collapse and expand
You can collapse the sidebar to icons‑only and expand it when you need labels.

> [!suggestion] What you should see
> - A narrower sidebar when collapsed.
> - Full hub labels when expanded.

## Content area (main)

The content area shows dashboards, functional cards, and full‑screen app views.

### Hero bar
The hero bar sits at the top of the content area to provide context for the current hub.

> [!suggestion] What you should see
> - A large banner with the hub name and supporting text.

### Functional cards
Cards are the primary way to launch tools and dashboards.

#### Card layout and positioning
- Cards appear in a grid within the current hub.
- Cards can be pinned to favourites (when enabled).

#### Opening a card
1. Select a card to open it.
2. A detailed view opens in the content area.

> [!suggestion] What you should see
> - A card detail view with the card title and description.
> - An **Open in new tab** option for external tools.

#### Card menu (if available)
Some cards include a menu to pin/unpin, hide, or adjust preferences.

> [!suggestion] What you should see
> - A menu icon on the card with actions such as pin or hide.

## Status bar (bottom)

The status bar provides a compact system view:
- **API health** indicators (e.g., Vault, PropertyMe).
- **Task banner** for overdue or due‑today tasks.
- **Notifications strip** with recent alerts.
- **User indicator** and app version.

### Task banner
1. If you have overdue or due‑today tasks, select the banner.
2. This opens the Tasks panel.
3. Dismiss to hide it temporarily.

> [!suggestion] What you should see
> - A banner with task counts and a tooltip on hover.

### API health
Health indicators show system connectivity at a glance.

> [!suggestion] What you should see
> - Green, amber, or red dots reflecting system status.

## Panels and flyouts

The UI uses slide‑out panels and flyouts for focused tasks:
- **Tasks panel** – manage tasks without leaving the hub.
- **Settings panel** – configure preferences.
- **Notification flyout** – review alerts.

> [!suggestion] What you should see
> - Panels appear on the right and close without full page navigation.

## Settings and preferences

Access settings from the user menu in the navbar.

Common settings include:
- Theme mode (light/dark).
- Sidebar behaviour.
- Personalisation preferences.

> [!suggestion] What you should see
> - A settings panel with sections and toggles.

## Tips for first‑time users

- Use the navbar search to quickly find tools.
- Pin frequently used cards to favourites (when available).
- Check the status bar for system alerts before starting work.
- Use the Tasks panel to track work across hubs.
